It is now possible to customize the editable element tag name, classes, styles, and attributes via `config.root.element` or `config.roots.main.element`. The value can be a plain string (tag name) or an object:

```html
<template>
    <ckeditor :editor="ClassicEditor" :config="config" />
</template>

<script setup>
import { ref } from 'vue';
import { ClassicEditor } from 'ckeditor5';

const config = {
    root: {
        element: {
            name: 'article',
            classes: [ 'my-editor', 'custom-class' ],
            styles: { color: 'red' },
            attributes: { role: 'textbox' }
        }
    }
};
</script>
```

For editors that use the Vue-rendered element as their editable (such as Inline or Balloon editor), the editable falls back to the `tagName` prop (`div` by default) when no element definition is provided in config. For Classic editor, which creates its own editable internally, `config.root.element` (or `config.roots.main.element`) should always be provided explicitly.
